Project overview

What is SourceInfi?

SourceInfi (by Shifto Pvt Ltd) is a full-stack drop-shipping and e-commerce platform connecting vendors who supply products with sellers who list and sell them on Shopify or WooCommerce stores — without holding inventory.

The platform handles the entire lifecycle: vendor catalog management, seller onboarding, pricing and margin rules, order ingestion via channel webhooks, shipment booking through multiple courier partners, real-time tracking, and wallet-based settlement.

Architecture approach

PHP + CodeIgniter + MySQL — seller operations, catalog, wallet, invoicing, and session-based auth. Monolithic core chosen for its transactional guarantees and rapid iteration.
Node.js + TypeScript + MongoDB — courier API orchestration, real-time event handling, and fast lookup of courier events. Microservice layer for I/O-bound courier calls.
RabbitMQ — decouples order ingestion, shipment tracking, and bulk import jobs. Exponential backoff and idempotent workers ensure resilience at scale.

Key modules

Seller & vendor registration

KYC verification, multi-tenant onboarding, wallet creation, and pricing plan assignment at sign-up.

Catalog management

Vendor product creation with SKUs, variants, cost pricing, and Cloudinary-hosted images. Bulk CSV import for sellers with markup rules.

Shopify app integration

OAuth-based channel connect, product push via Admin API, real-time order webhook ingestion, and delivery status sync-back.

Pincode & SLA management

Serviceability checks per pincode, delivery SLA computation per origin–destination pair for smarter courier selection.

Shipment process

AWB generation, dynamic courier allocation, label creation via S3, and batch shipment processing through NimbusPost and NimbusLM APIs.

Shipment tracking

Dual-mode tracking: cron polling for all undelivered shipments and real-time courier webhook push (Ekart, Shipway). RabbitMQ queues decouple updates.

NDR handling

Automated non-delivery report workflows, rescheduling flows, and push to courier partners with exponential retry logic and seller notifications.

Wallet & margin management

Seller wallets with debit/credit ledger, margin adjustments, COD reconciliation, RTO charge reversals, and monthly consolidated invoicing.

Architecture decisions

Hybrid monolith + microservices — PHP/CI handles transactional seller ops (ACID guarantees), Node.js handles high-throughput courier I/O where event-loop concurrency wins.
Event-driven queue design — RabbitMQ exchanges for order ingestion, shipment tracking, and bulk imports decouple producers from consumers and enable retry without data loss.
Dual-mode tracking — cron polling covers couriers without webhook support; real-time webhook push (Ekart, Shipway) reduces status lag. Both converge into a single status update path.
Idempotent wallet ledger — each debit/credit event is keyed by AWB + transaction type, preventing duplicate entries during retry storms and reconciliation runs.
