Project Overview

Dropship is a procurement & fulfillment platform enabling merchants to route orders to suppliers, track shipments at box-level, and reconcile receipts with GRNs. The platform produces business-ready PI/Shipment exports and automates catalog & PI syncs to reduce manual work.

My Role

Sole developer — responsible for requirements analysis, system architecture, UI design, backend development (Laravel), job/observer-based sync automation, and export formatting. Led manual QA and supported initial production rollout.

Key Challenges & Solutions

1. Multi-box reconciliation

Refactored payload generation to compute latest-GRN-per-box and emit per-box rows plus a single "remaining" row per product. Eliminated duplicated "new" rows in exports.

2. Cross-collection filters with correct pagination

Precomputed PO ID and catalog ID filters and applied them before paginating to ensure page results matched global filters.

3. Robust external syncs

Implemented observer → queued job flow for catalog and PI syncs (switchable from sync to async driver). Added retries and detailed logs for failure diagnosis.
